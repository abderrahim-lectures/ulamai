# `.ulam/config.json` reference

JSON doesn't support comments, so this file documents every field instead
of inline `//` comments in `config.json` itself. Generated by reading the
`_resolve_*` functions in `ulam/cli.py` that actually consume each field —
if a field isn't listed here, nothing currently reads it.

For every field below: **an explicit CLI flag always wins over config**;
config is only the fallback when no flag is passed.

## `llm_provider`

Which LLM adapter `ulam prove`/`formalize` use by default (`--llm` overrides).
One of: `openai`, `anthropic`, `gemini`, `gemini_cli`, `ollama`, `codex_cli`,
`claude_cli`.

## `llm` — runtime controls (shared by every provider)

| Field | Type | Meaning |
|---|---|---|
| `timeout_s` | number | Per-LLM-call timeout in seconds. `0` = no timeout. |
| `heartbeat_s` | number | Heartbeat/progress-log interval while waiting on a call. |

## `openai` / `anthropic` / `gemini` / `embed` — provider settings

Each has `api_key`, `base_url`, `model` (and `anthropic` also has
`setup_token`, `codex_model`/`claude_model`/`cli_model` for the matching
subscription-CLI backend). Env var equivalents exist for each — see
`.env.example`. `embed` is used by `ulam index build`'s embedding retriever.

## `ollama`

| Field | Default | Meaning |
|---|---|---|
| `base_url` | `http://localhost:11434` | Ollama server URL. |
| `model` | `llama3.1` | Must be a model you've actually pulled (`ollama list`) — an unpulled `:latest` tag 404s. |

## `prove` — `ulam prove` defaults

| Field | Type / choices | Default | Meaning |
|---|---|---|---|
| `mode` | `tactic`\|`lemma`\|`llm` | `tactic` | Proof search mode. Also what `formalize.proof_backend: "inherit"` falls back to. |
| `output_format` | `lean`\|`tex` | `lean` | `lean` = machine-checked pipeline; `tex` = informal planner/worker/judge pipeline. |
| `solver` | `auto`\|`search`\|`script`\|`portfolio` | `auto` (→ `script` if mode is `lemma`, else `search`) | Search strategy. |
| `allow_axioms` | bool | `true` | Allow `axiom`/`constant` in accepted proofs. |
| `typecheck_timeout_s` | number, min 5 | `60` | Per-typecheck timeout. |
| `llm_allow_helper_lemmas` | bool | `true` | Let the LLM introduce helper lemmas (mode `llm`). |
| `llm_edit_scope` | `full`\|`errors_only` | `full` | Mode `llm`: rewrite the whole file or only error spans. |
| `llm_cycle_patience` | int, min 1 | `2` | Mode `llm`: rounds to tolerate no progress before giving up. |
| `search_lean_backend` | `dojo`\|`cli`\|`lsp` | `dojo` | Lean backend for tactic/lemma search. |
| `lemma_max` / `lemma_depth` | int | `60` / `60` | Lemma-first planning limits. |
| `retriever_k` / `retriever_source` / `retriever_build` / `retriever_index` | — | `8` / `local` / `auto` / `""` | Premise retrieval settings. |
| `tex_rounds` | int, min 1 | `3` | Output format `tex`: whole-theorem attempt rounds. |
| `tex_judge_repairs` | int, min 0 | `2` | Output format `tex`: judge-requested repair attempts. |
| `tex_worker_drafts` | int, min 1 | `2` | Output format `tex`: parallel worker drafts per claim. |
| `tex_concurrency` | bool | `false` | Output format `tex`: solve claims concurrently. |
| `tex_replan_passes` | int, min 1 | `2` | Output format `tex`: replan/backtrack passes when stalled. |
| `tex_action_steps` | int, min 1 | `10` | Output format `tex`: bounded planner-action loop steps. |
| `tex_planner_model` / `tex_worker_model` | string | `""` (use main provider) | Split planner vs. worker models. |
| `tex_out_dir` | path | `proofs` | Where composed `.tex` proofs are written. |
| `tex_artifacts_dir` | path | `runs/prove_tex` | Resumable run artifacts (`state.json`, `events.jsonl`, ...). |

## `formalize` — `ulam formalize` defaults

| Field | Type / choices | Default | Meaning |
|---|---|---|---|
| `proof_backend` | `tactic`\|`lemma`\|`llm`\|`dojo`\|`inherit` | `inherit` | Per-declaration proof search backend. `dojo` is a CLI-only alias for `tactic`. `inherit` = use `prove.mode`. |
| `lean_backend` | `dojo`\|`cli`\|`lsp` | `dojo` | Typecheck backend. Note: if `proof_backend` resolves to `llm` and this is `dojo`, `ulam` auto-downgrades it to `cli` (the `llm` proof backend doesn't use Dojo's goal state). |
| `max_rounds` | int, min 1 | `5` | Draft/repair rounds before giving up. |
| `max_repairs` | int, min 0 | *(falls back to `max_rounds` if unset)* | Repair attempts across all rounds. |
| `max_equivalence_repairs` | int | `2` | Repairs when the equivalence checker flags a statement drift. |
| `max_proof_rounds` | int, min 1 | `1` | Per-declaration proof-search rounds. |
| `proof_repair` | int, min 0 | `2` | Per-declaration proof repair attempts. |
| `typecheck_timeout_s` | number, min 5 | `60` | Per-typecheck timeout. |
| `llm_check` | bool | `true` | Run the LLM semantic-integrity audit. |
| `llm_check_timing` | `end`\|`mid+end` | `end` | When the semantic audit runs. |
| `llm_check_repairs` | int, min 0 | `2` | Repair attempts when the semantic audit fails. |

## `segmentation`

| Field | Default | Meaning |
|---|---|---|
| `chunk_words` | `1000` | Max words per `.tex` segment when splitting long documents. |

## `lean`

| Field | Type | Meaning |
|---|---|---|
| `project` | path | Default Lean project root (`--lean-project` overrides). **Use an absolute path** — a relative one resolves against whatever cwd `ulam` happens to be invoked from. |
| `imports` | list[str] | Extra Lean imports injected into every proving session. |
| `dojo_timeout_s` | number | Pantograph/LeanDojo server startup+call timeout. |

## `policy`

| Field | Choices | Default | Meaning |
|---|---|---|---|
| `proof_profile` | `fast`\|`balanced`\|`strict` (also accepts `normal`/`default` as aliases for `balanced`) | `balanced` | Guardrail budget profile trading speed vs. verification depth. `--proof-profile` overrides. |
